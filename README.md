## About me

- Building low-latency backend systems and infra-heavy tools.
- Interested in storage engines, search, indexing, and practical distributed systems.
- Mostly working in Go, with a focus on performance, ergonomics, and predictable behavior under load.

## What I'm working on

- [`unch`](https://github.com/uchebnick/unch) - local-first semantic code search for repository annotations.
- [`fusedb`](https://github.com/uchebnick/fusedb) - embedded KV engine for small hot mutable state with local leaf-level merge.
- [`ton-storage-s3-cli`](https://github.com/uchebnick/ton-storage-s3-cli) - S3-compatible CLI for TON Storage written in Go.
- [`unch-docs`](https://github.com/uchebnick/unch-docs) - documentation for `unch`.
